# 第三部分导读：系统设计方法

> 本部分回答：**如何用系统工程方法，把各个子系统集成为一个协调、可靠、可验证的整体？**

## 学习目标

读完本部分，你应该能够：

1. 用六层架构和 V 模型组织研发流程
2. 把船东的模糊需求分解为量化的工程需求
3. 建立多物理域协同仿真体系，在设计阶段暴露问题
4. 用 ICD 管理子系统间接口，用 RAMS 保证可靠性

## 工程要点

- **风险前置**：不等到海试再发现问题，设计阶段就识别并处理
- **接口是事故高发区**：子系统各自没问题，集成时出问题——80% 的集成问题源于接口定义不清
- **仿真省钱**：台架试验改一次几十万，仿真改一次零成本
