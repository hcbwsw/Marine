## 3.1 MBSE 五视图体系

### 视图一：需求图

```
[REQ] 船用电驱系统
  ├── [REQ-001] 推力性能：额定推力 ≥ XX kN @设计航速
  ├── [REQ-002] 效率：推进效率 ≥ 92%（额定工况）
  ├── [REQ-003] 热管理：绕组温升 ≤ 80K（F级，连续运行）
  ├── [REQ-004] 可用性：A ≥ 99.5%（年统计）
  ├── [REQ-005] 噪声：机舱噪声 ≤ 85dB(A)（测量距离1m）
  ├── [REQ-006] 安全：单一故障不导致推力完全丧失
  └── [REQ-007] 合规：满足 CCS/DNV 适用章节，通过船级社审图
```

### 视图二：功能架构

```
F0：产生满足任务需求的推力
  ├── F1：将电能转化为旋转机械能    → 传动子系统（电机）
  ├── F2：将旋转机械能转化为推力    → 螺旋桨
  ├── F3：维持系统热平衡            → 冷却子系统
  ├── F4：精确控制转速/转矩         → 控制子系统（FOC）
  ├── F5：传递轴向推力至船体        → 推力轴承+船体推力块
  ├── F6：监测系统健康状态          → 传感器组+控制器
  └── F7：故障保护与降级运行        → 控制逻辑+冗余设计
```

**功能分配准则（每个功能必须有且仅有一个主责子系统）：**

```
F1（电能→机械能）：主责=传动子系统，支持=控制子系统（FOC指令）
F3（热平衡）：主责=冷却子系统，触发=控制子系统（泵调速）
F4（转速控制）：主责=控制子系统，被控=传动子系统
F7（故障保护）：主责=控制子系统，执行=三个子系统协同

⚠️ 若某功能"主责不清"（两个子系统都认为对方负责）
   → 该功能在系统集成时极可能出现遗漏（典型集成故障根因）
```

### 视图四：行为视图（热保护状态机）

```
[正常运行] T_winding ≤ 130°C
      │ T > 130°C（持续5s）
      ▼
[一级热警告] 泵升速至100%，记录告警
      │ T > 155°C（持续10s）              │ T < 125°C（持续60s）→ [正常运行]
      ▼
[二级降额运行] P = P_rated×(170-T)/(170-155)
      │ T > 175°C                         │ T < 145°C（持续120s）→ [一级热警告]
      ▼
[紧急停机] 切断驱动，保留冷却泵散热，需人工复位
```

**完整系统状态机（含冷却故障并发状态）：**

```
                    ┌─────────────────┐
                    │   正常运行       │
                    │  T<130°C        │
                    │  流量>70%额定   │
                    └────────┬────────┘
              ┌──────────────┼──────────────┐
              ▼              ▼              ▼
        [热预警]        [冷却降级]      [传感器异常]
        T>130°C        流量<70%额定    传感器信号失效
        泵全速          自动降额至80%   切换冗余/估算
              │              │              │
              └──────────────┼──────────────┘
                             ▼
                      [降额运行]
                      P=40~80%额定
                             │ 条件继续恶化
                             ▼
                      [紧急停机]
                      T>175°C 或 流量<30%额定
                      需人工复位
```

### 视图五：参数视图（关键参数约束关系）

```
约束1：热功率平衡
  P_cooling ≥ P_loss_total × 1.3（30%裕度）
  P_loss = 3I²R + K_h×f×B²×V_core + K_e×f²×B²×V_core + P_mech

约束2：泵工作点约束
  Q_pump(H) 与 Q_system(H) 的交点即为工作点
  工作点须在泵效率曲线 η_pump ≥ 70% 区间内
  📐 验证方法：绘制泵Q-H曲线与系统阻力曲线，确认工作点位置

约束3：轴系临界转速约束
  n_critical(正) ≥ 1.2 × n_max
  n_critical(逆) ≤ 0.8 × n_min
  📐 验证方法：扭振分析报告 + Campbell图（船级社审查项）

约束4：控制环带宽约束
  f_current ≥ 10 × f_torsion（电流环带宽远高于扭振频率）
  f_speed ≤ f_torsion / 3（速度环带宽低于扭振频率1/3）
  📐 验证方法：FOC参数整定报告 + 台架扭振验证

约束5：RAMS可用性约束
  A = MTBF / (MTBF + MTTR) ≥ 99.5%
  由各子系统 MTBF 串并联计算决定
  📐 验证方法：RAMS分析报告（见第12章）
```

## 3.2 公理化设计耦合诊断与解耦方案

**完整设计矩阵：**

```
{FR1}   [X 0 0 0 0] {DP1}   电机功率×螺旋桨匹配
{FR2} = [x X 0 0 0] {DP2}   FOC带宽+编码器分辨率
{FR3}   [X 0 X 0 0] {DP3}   换热面积+冷却流量
{FR4}   [x 0 0 X 0] {DP4}   推力轴承额定载荷
{FR5}   [x x x x X] {DP5}   冗余策略

矩阵为下三角形 → 解耦设计（Decoupled Design），可接受
（若为全矩阵 → 耦合设计，需要重新架构）
```

> ⚠️ **FR3 与 DP1 存在强耦合**（电机损耗直接决定热源强度），冷却系统必须与传动系统协同设计，不可独立迭代。这是最常被忽视的系统级耦合！

**FR3 与 DP1 强耦合的解耦方案：**

```
问题根源：电机功率越高 → 铜耗越大 → 热源越强 → 冷却要求越高
          这是物理耦合，无法消除，只能管理

解耦策略：
  ✅ 选项A（推荐）：前馈解耦
     Q_pred = P_cmd × (1 - η_motor)  ← 根据功率指令预测热源
     提前调节泵速，热响应滞后从 ~60s 降至 ~5s
     实现：控制器读取功率指令 → 计算预期损耗 → 前馈调节泵转速

  ✅ 选项B：参数化冷却能力
     按功率档位预设冷却液流量（如30%/60%/100%三档）
     优点：控制简单；缺点：非连续有跳变（可能影响温控平稳性）

  ✅ 选项C（保底）：降额律截止
     承认耦合，靠降额律在边界处保护
     必须配合选项A或B，不能单独使用

推荐组合：A（前馈主动预控）+ C（降额律保底）

⚠️ 仅使用选项C的风险：
   热响应滞后 → 绕组温度已超限 → 降额 → 热负荷降低 →
   温度恢复 → 功率恢复 → 温度再超限 → 周期性振荡（"热振荡"）
   典型表现：功率输出周期性波动，船速不稳定，乘客投诉
```

## 3.3 架构权衡研究（Trade Study）

### Trade Study 1：冷却拓扑方案选择

| 评估维度 | 权重 | 单回路海水直冷 | 双回路（推荐）| 双回路+备泵冗余 |
|---------|------|-------------|-------------|---------------|
| 可靠性 | 25% | 2 | 4 | 5 |
| 维护性 | 20% | 3 | 4 | 4 |
| 成本 | 20% | 5 | 3 | 2 |
| 重量/体积 | 15% | 5 | 3 | 2 |
| 电机保护 | 20% | 2 | 5 | 5 |
| **加权总分** | — | **3.25** | **3.90** | **3.70** |

> **结论：双回路方案综合最优。海船/客船（A≥99.9%要求）加装备泵冗余。**

### Trade Study 2：电机直驱 vs 减速器方案

| 评估维度 | 权重 | 直驱方案 | 减速器方案 |
|---------|------|---------|---------|
| 传动效率（额定）| 25% | 5（无减速损耗）| 3（减速器效率约97%）|
| 系统复杂度 | 20% | 5（部件少）| 2（减速器+润滑系统）|
| 维护工作量 | 15% | 5（无齿轮维护）| 2（定期换油/齿轮检查）|
| 电机功率密度 | 20% | 2（低转速=大体积）| 4（高转速电机更紧凑）|
| 成本 | 10% | 3（大功率低速电机较贵）| 3（减速器+小电机）|
| 噪声/振动 | 10% | 4（无齿轮啮合）| 2（齿轮噪声）|
| **加权总分** | — | **3.80** | **2.85** |

> **结论：直驱方案综合优势明显。仅在螺旋桨需极低转速（<100rpm）时考虑减速器。**

### Trade Study 3：定距桨（FPP）vs 可调桨（CPP）

| 评估维度 | 权重 | 定距桨（FPP）| 可调桨（CPP）|
|---------|------|-----------|-----------|
| 设计点推进效率 | 25% | 5（高）| 4（桨毂损失约2%）|
| 低速/靠泊效率 | 15% | 2（需电机变速）| 5（桨距可调，效率高）|
| 系统复杂度 | 20% | 5（简单）| 2（液压系统+控制）|
| 成本 | 20% | 5（低）| 2（约2~3倍FPP）|
| 与变频器配合 | 10% | 3（需全范围变频）| 4（可恒速运行）|
| 维护工作量 | 10% | 5（无可动部件）| 2（液压系统维护）|
| **加权总分** | — | **4.10** | **3.10** |

> **结论：固定航线首选定距桨+变频直驱。复杂工况（港作拖轮、多档位频繁切换）可考虑CPP。**

### Trade Study 使用注意事项

```
Trade Study 执行规范：
  ① 评估维度和权重必须在权衡开始前由团队协商确定，不得事后修改
  ② 所有方案的评分由独立评审人完成（避免主观倾向）
  ③ Trade Study 结论必须记录在设计决策记录（DDR）中
  ④ 评分为1~5分：5=最优，1=最差（相对评分）

❌ 常见错误：
   权重选择暗含对某方案的偏好 → 结论失去公信力
   评分在知道结论后反向填写 → 完全无效的Trade Study

✅ 正确做法：
   先定评估维度和权重（PDR前）→ 盲打分 → 计算结果 → 归档DDR
```

**N² 矩阵使用规则：**

```
① N²矩阵每个非空格=一份ICD（接口控制文件）
   上表共有 ~12个非空接口 → 需编写12份ICD

② 每份ICD必须有双方（提供方+接收方）的责任工程师签字
   ICD Rev.A（架构阶段草稿） → ICD Rev.B（CDR时冻结）

③ N²矩阵是动态文件：
   每次发现新接口 → 更新N²矩阵 → 新增ICD
   接口变更 → ECR/ECO流程 → 更新对应ICD

④ 集成问题80%源于接口未定义或定义不清楚
   N²矩阵完整 = 集成风险可控的第一步
```

## 3.5 冗余架构设计策略

```
船用电推系统冗余需求（REQ-006：单一故障不导致推力完全丧失）：

系统级冗余实现方式（选其一或组合）：

方案A：双轴推进
  两套独立电驱系统（各含独立冷却+传动+控制）
  单轴故障 → 剩余单轴维持50%~70%推力
  适用：对称双轴船（大多数商船）
  成本：最高（两套完整系统）

方案B：单轴+侧推器应急
  主推进单轴 + 应急侧推器
  主轴故障 → 侧推维持操纵性（推力有限）
  适用：空间受限的单轴船、渡船

方案C：N+1 冗余（组件级）
  单轴推进 + 关键组件备份（双泵/双控制器/双编码器）
  组件故障 → 降级运行（降额后单组件维持）
  适用：内河船（风险等级较低）
  成本：最低

方案D：功能降级策略
  推进功率按P1/P2/P3三档设计（100%/70%/40%）
  故障后降至下一档运行，完全停机为最后手段
  与方案A/B/C结合使用

⚠️ 冗余的代价：重量/体积/成本增加
   正确做法：用RAMS分析确认冗余效果量化收益 vs 代价
   而非"感觉要冗余"的主观判断
```

## 3.6 本章小结与关键输出物

**本章核心要点（四条）：**

1. **MBSE五视图是架构设计的完整工具集**
   需求图→功能架构→逻辑架构→行为视图→参数视图，五步缺一不可。行为视图（状态机）是控制逻辑设计的直接输入。

2. **公理化设计矩阵诊断耦合，下三角形是合格线**
   FR3（热管理）与DP1（电机功率）的强耦合是本系统最大设计挑战，前馈解耦是唯一工程可行的主动控制方案。

3. **Trade Study 是架构决策的工程依据，不是走形式**
   权重和评分必须独立于结论确定，结论归档DDR，方案迭代时重新执行，不得"改权重凑结论"。

4. **N²矩阵决定集成成败**
   12个接口=12份ICD，ICD Rev.B在CDR时冻结。任何CDR后的接口变更都需要走重大变更（ECR/ECO）流程。

**关键输出物：**
```
□ MBSE五视图文件（含状态机图和参数约束表）
□ 公理化设计矩阵（FR/DP对应关系，耦合分析）
□ 三份Trade Study报告（冷却拓扑/直驱减速器/FPP-CPP）
□ N²接口矩阵（完整版，所有接口识别完整）
□ 12份ICD Rev.A（架构阶段草稿，双方责任工程师确认）
□ 设计决策记录DDR（所有架构决策，含被否决方案及理由）
□ CDR强制检查单（15条全绿）
```
