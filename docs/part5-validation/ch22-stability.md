

# 第21章 稳定运行相图与降级走廊

> **本章导读**
> 稳定运行相图是船用电推系统"能力边界的可视化地图"——
> 它回答一个核心工程问题：**在任意外部环境条件和系统健康状态下，
> 系统当前最多能输出多少功率，且保证安全不停机？**
>
> 降级走廊则是相图的动态执行机制——
> 当系统从绿区漂移向红区时，控制器沿"走廊"有序退出，
> 而不是突然崩溃停机。
>
> 本章从相图构建原理出发，给出完整的降额律公式推导、
> 阈值整定方法、控制实现要点、标定流程和典型案例，
> 是第5章热保护逻辑（6.3节）和第11章接口管理（ICD-CD-001）的
> 工程落地细化。

---

## 10.1 稳定运行相图——概念与构建

### 10.1.1 为什么需要相图

```
传统做法（单变量保护）的局限性：
  仅监测绕组温度T_winding，超过阈值就停机
  
  问题：
  ① 单一变量无法反映多因素耦合：
     同样T_winding = 140°C，
     原因A：海水温度突然升高（可通过降额继续运行）
     原因B：冷却液流量突降（须立即降额+检查）
     原因C：系统已处于热平衡（可稳定在此温度运行）
     → 三种情况处置方式完全不同，单变量无法区分

  ② 保守设计造成浪费：
     为应对最恶劣工况（T_sea=35°C+换热器污堵50%），
     将额定功率降至保守值，导致正常工况大量性能浪费

  ③ 无法给运维人员提供直观指导：
     运维人员不知道"当前系统还能承受多少负荷"

相图的价值：
  ① 多维可视化：同时展示温度、压损、功率三个维度
  ② 预测性：根据当前工况点位置预判趋势（向红区漂移的速度）
  ③ 主动性：在到达红区之前，沿降级走廊有序退出，而非被动停机
  ④ 可解释性：驾驶台HMI直观显示当前工况点，运维人员一眼判断状态
```

### 10.1.2 相图坐标系定义（完整版）

```
主相图（二维投影，控制台显示用）：

  X轴：外回路压损增量 Δp_ext（%）
    定义：当前外回路压降 vs 初始清洁状态压降的增幅百分比
    计算：Δp_ext = (p_current - p_baseline) / p_baseline × 100%
    物理含义：换热器/滤器污堵程度的间接表征
    测量：压差传感器（安装于换热器两端），分辨率 ≤ 0.5kPa
    范围：0%（全新清洁）~ 100%（严重污堵，接近阻塞）

  Y轴：海水温度 T_sea（°C）
    定义：外循环换热器海水侧入口温度
    测量：PT100传感器（安装于海水进管），精度 ±0.5°C
    范围：-5°C（北方冬季极值）~ 40°C（热带极端海况）
    设计基准：35°C（CCS/DNV 通常取值）

  Z轴（颜色填充）：系统可输出功率 P_available（%额定）
    定义：在当前（T_sea, Δp_ext）条件下，
          系统可稳定连续输出的最大功率，不触发热保护
    计算：由降额律公式实时计算（见10.2节）
    颜色编码：
      ■ 深绿（P ≥ 95%）：正常满功率运行区
      ■ 浅绿（P = 80~95%）：轻度受限，可接受
      ■ 黄色（P = 60~80%）：中度受限，需关注冷却系统
      ■ 橙色（P = 40~60%）：重度受限，降额运行，尽快维护
      ■ 红色（P < 40%）：禁止运行区，强制停机保护

  辅助显示元素：
    ★ 当前工况点（实心圆点，实时更新）
    → 趋势箭头（过去10min的漂移方向和速度）
    ── 等功率线（P=100%/80%/60%/40%的等值线轮廓）
    ▲ 历史最高温度点（本次航行记录）
    虚线框：设计工况点（T_sea=35°C，Δp=0%，P=100%）

三维相图（工程分析用，非实时显示）：
  在主相图基础上增加第三维：内循环液入口温度T_in
  形成三维曲面P = f(T_sea, Δp_ext, T_in)
  用途：台架标定、仿真验证、边界案例分析
  工具：MATLAB surf函数 或 Python matplotlib 3D
```

### 10.1.3 相图分区边界的物理推导

```
分区边界线的推导（以绿区/黄区边界为例）：

  绿区/黄区边界 = P_available = 80% 额定功率的等值线

  条件：在该边界上，绕组温度恰好达到T_warn = 130°C（一级报警）
        即：热平衡方程在T_winding = 130°C时成立

  热平衡方程：
    Q_loss(P) = Q_removed(T_sea, Δp_ext, T_in)

    其中：
    Q_loss(P) = P_rated × η_loss × (P/P_rated)^α
      η_loss ≈ 4~8%（损耗率，依电机效率）
      α ≈ 1.5~2（铜耗主导时α=2，铁耗+铜耗混合时α<2）

    Q_removed = m_dot_cool × Cp × (T_out - T_in)
              = ε × C_min × (T_sea - T_in)  ← ε-NTU法

    其中换热效能ε随Δp_ext下降（污堵→换热恶化）：
    ε(Δp_ext) = ε_0 × (1 - k_foul × Δp_ext / 100)
      ε_0 = 初始换热效能（清洁状态，典型0.85~0.90）
      k_foul = 污堵敏感系数（经验值0.3~0.5，需台架标定）

  联立以上方程，在T_winding = 130°C约束下，
  可解出每个(T_sea, Δp_ext)对应的P_boundary
  → 扫描T_sea × Δp_ext的网格 → 得到边界曲线

  边界曲线特征（定性）：
    T_sea低时：边界线Δp_ext值高（温度低，可承受更大污堵）
    T_sea高时：边界线Δp_ext值低（温度高，污堵裕度小）
    → 边界线从左下到右上单调递减（负斜率）
```

## 10.2 降额律参数化公式（完整推导版）

### 10.2.1 多因子降额模型

$$P_{\text{derate}}(T_{\text{sea}}, \Delta p, T_{\text{in}}) = P_{\text{rated}} \times K_1(T_{\text{sea}}) \times K_2(\Delta p) \times K_3(T_{\text{in}})$$

**三个降额因子的物理含义与推导：**

```
K₁(T_sea)——海水温度因子：
  物理含义：外部热汇温度升高，换热温差减小，冷却能力下降
  
  推导：换热器移热量 Q = ε × C_min × (T_sea - T_in)
        T_sea升高 → (T_sea - T_in)减小（若T_in由T_sea决定，差值缩小）
        → 冷却能力下降 → 允许热耗（即允许功率）下降
  
  分段公式：
    K₁ = 1.0                        当 T_sea ≤ 25°C（北温带，冷却充裕）
    K₁ = 1 - 0.012×(T_sea - 25)    当 25°C < T_sea ≤ 35°C（线性降额）
    K₁ = 0.88                       当 T_sea > 35°C（超出设计范围，固定限额）
  
  关键数值：
    T_sea = 30°C：K₁ = 0.94（降额6%）
    T_sea = 35°C：K₁ = 0.88（降额12%，标准设计基准点）
    T_sea = 40°C：K₁ = 0.88（触发K₃进一步限制）
  
  注：K₁系数斜率0.012为工程估算，实际斜率须通过台架试验
      在不同T_sea条件下测量热平衡功率后回归确定

─────────────────────────────────────────────────

K₂(Δp)——外回路污堵因子：
  物理含义：换热器/滤器污堵 → 外回路压降增加 → 
            ① 换热效率下降（污堵层热阻增加）
            ② 外循环泵工作点偏移（流量减少）
            → 双重冷却能力退化
  
  分段公式：
    K₂ = 1.0                        当 Δp ≤ 20%（轻度污堵，可忽略）
    K₂ = 1 - 0.008×(Δp - 20)      当 20% < Δp ≤ 50%（中度污堵，线性降额）
    K₂ = 0.76                       当 Δp > 50%（重度污堵，固定限额+强制清洗提醒）
  
  关键数值：
    Δp = 30%：K₂ = 0.92（降额8%）
    Δp = 50%：K₂ = 0.76（降额24%，触发维护提醒）
    Δp > 50%：保持0.76并向驾驶台发送"换热器清洗必要"报警
  
  实施注意：
    Δp_baseline需在每次清洗后重新标定
    控制器维护菜单提供"压损基准值重置"功能（操作员确认后执行）

─────────────────────────────────────────────────

K₃(T_in)——绕组温度实时反馈因子：
  物理含义：K₁×K₂是前馈（基于环境参数预测），
            K₃是反馈（基于绕组实测温度修正）
            当前馈估算误差较大（如热模型偏差）时，K₃兜底保护
  
  公式：
    K₃ = 1.0                                    当 T_in ≤ 45°C（冷却液正常）
    
    K₃ = (T_shutdown - T_winding,max) /         当 T_in > 45°C 且
         (T_shutdown - T_derate)                     T_winding > T_derate
    
    其中：
      T_shutdown = 175°C（H级绝缘停机阈值）
      T_derate   = 155°C（降额启动阈值）
      T_winding,max = 三路PT100投票取中间值（见ICD-DC-002）
  
  实际效果：
    T_winding = 155°C：K₃ = (175-155)/(175-155) = 1.0（刚触发降额，不额外限制）
    T_winding = 165°C：K₃ = (175-165)/(175-155) = 0.5（降额至50%）
    T_winding = 175°C：K₃ = 0 → 触发停机（由独立停机逻辑处理，不经过K₃）
```

### 10.2.2 降额律的工程实现

```
控制器实现（伪代码）：

  函数 compute_derating():
    // 读取传感器
    T_sea   = read_sensor(SEA_WATER_TEMP)        // PT100，外循环入口
    delta_p = compute_pressure_increase()         // (p_now - p_baseline) / p_baseline
    T_wind  = median(T_wind_1, T_wind_2, T_wind_3) // 三路投票取中间值
    
    // 计算三个降额因子
    K1 = calc_K1(T_sea)
    K2 = calc_K2(delta_p)
    K3 = calc_K3(T_wind)
    
    // 综合降额功率
    P_available = P_rated × K1 × K2 × K3
    
    // 平滑处理（防止降额指令突变冲击轴系）
    P_available_smoothed = rate_limiter(P_available, 
                                         ramp_down = 10%/s,  // 降额速率限制
                                         ramp_up   = 2%/s)   // 恢复速率更慢（防振荡）
    
    // 最小功率保护（防止降额至0导致船失去推力）
    P_output = max(P_available_smoothed, P_minimum)
    // P_minimum = 15%额定（维持最低航速，安全进港）
    
    // 向驾驶台报告
    report_to_bridge(P_available, K1, K2, K3, derating_reason_code)
    
    return P_output

计算周期：100ms（Level 2控制器，足够响应热时间常数>>1s）
执行位置：Level 2推进控制器（PDS Controller，见6.1节）
输入信号：通过ICD-DC-001/ICD-CS-002接口获取
输出信号：P_available（百分比）→ 通过ICD-CD-002下发给变频器
```

### 10.2.3 降额因子标定流程

```
台架标定试验（必须执行，不可跳过）：
  原则：K₁/K₂系数为理论估算，实际系数须通过台架试验标定
  标定误差目标：P_available计算值 vs 实测热平衡功率偏差 ≤ ±5%

  K₁标定试验：
    前提：换热器处于清洁状态（Δp = 0，K₂ = 1）
    方法：
      通过冷却水箱控制T_sea分别在20/25/30/35°C四个工况
      每个工况：逐步升高电机功率，直至绕组温度达到130°C（稳态）
      记录此时的功率值 = P_available_measured
    标定：
      K₁_actual(T_sea) = P_available_measured / P_rated
      与公式值K₁_formula对比，若偏差>5%则修正公式斜率

  K₂标定试验：
    前提：T_sea固定在设计值35°C
    方法：
      在外循环管路上串联节流阀，手动增加压降模拟污堵
      分别在Δp = 0/20%/35%/50%四个工况
      每个工况：升功率至绕组130°C，记录P_available
    标定：
      K₂_actual(Δp) = P_available_measured / (P_rated × K₁(35°C))
      与公式值对比，修正k_foul系数

  K₃验证试验（HIL注入）：
    方法：在实物台架上通过HIL注入偏置温度信号
          模拟T_winding = 155/160/165/170°C的降额过程
    验证：控制器按公式正确执行K₃降额，功率响应曲线符合预期
    无需修改公式（K₃基于实测温度反馈，无需标定）

  标定结果归档：
    标定报告编号：CAL-DERATE-001
    在VCRM中登记（REQ-CTRL-007降额精度需求的验证证据）
    标定后的K₁/K₂参数存入控制器EEPROM（掉电不丢失）
```

## 10.3 降级走廊——完整设计

### 10.3.1 降级走廊的概念

```
"走廊"的比喻：
  想象系统运行状态是一个人在相图中行走
  绿区 = 宽阔大厅（自由行走）
  黄区 = 走廊（受约束，但有出口）
  橙区 = 紧急通道（限制通行，尽快撤离）
  红区 = 禁止区域（触发硬件保护，立即停机）
  
  "降级走廊"的设计目标：
    当系统从绿区向红区漂移时，
    控制系统主动引导系统沿走廊向安全方向移动，
    而不是被动等待碰到红区边界后紧急停机
    
    主动降级 vs 被动停机：
    ┌──────────────┬────────────────┬────────────────┐
    │ 对比项       │ 主动降级（走廊）│ 被动停机       │
    ├──────────────┼────────────────┼────────────────┤
    │ 推力保留     │ 保留15~70%推力  │ 推力完全丧失   │
    │ 恢复时间     │ 可快速恢复      │ 需手动复位     │
    │ 船员体验     │ 平稳，可提前告知│ 突然，可能慌乱 │
    │ 轴系冲击     │ 斜坡降额，无冲击│ 急停，冲击大   │
    │ 乘客感受     │ 轻微减速        │ 突然停推       │
    └──────────────┴────────────────┴────────────────┘
```

迟滞设计原则（防止在阈值附近反复触发）：
  恢复阈值 = 触发阈值 - ΔT_hyst（迟滞死区）
    L1温度：ΔT_hyst = 5°C，时间迟滞60s
    L2温度：ΔT_hyst = 10°C，时间迟滞120s
    流量：ΔQ_hyst = 10%额定，时间迟滞30s
    压损：Δp_hyst = 20%基准值，时间迟滞300s
  
  时间迟滞的必要性：
    若无时间迟滞：温度在阈值附近±1°C波动 → 降额/恢复反复切换
                  → 功率输出震荡 → 轴系扭矩冲击 → 乘客不适
    有时间迟滞：温度必须稳定低于恢复阈值持续60/120s才解除降额
                → 确认冷却系统已真正稳定，而非瞬时波动
```

### 10.3.3 降级走廊状态机

```
状态转移条件详表（完整版）：

  S0 → S1：收到推进指令
            AND 系统自检通过（冷却流量正常 + 无未复位故障）
            AND P_available ≥ 40%额定（当前环境条件允许）

  S1 → S2：任一 L1 条件触发：
            绕组温度 > 130°C
            OR 冷却液流量 < 60%额定
            OR 外回路压损 > 150%基准值

  S2 → S1：所有 L1 条件同时满足恢复阈值
            （温度 < 125°C 持续 60s，流量 > 70% 持续 30s，
             压损 < 130% 持续 300s）

  S2 → S3：任一 L2 条件触发：
            绕组温度 > 155°C（启动 K₃ 线性降额）
            OR 冷却液流量 < 60%额定持续超过 30s（未自愈）
            OR P_available 计算值 < 80%额定

  S3 → S2：所有降额因子恢复（K₁×K₂×K₃ ≥ 0.95）
            且绕组温度 < 145°C 持续 120s（迟滞保护）

  S3 → S4：P_available 计算值 < 40%额定

  S4 → S5：任一停机条件触发：
            绕组温度 > 175°C（L3 停机）

            OR 膨胀箱极低液位信号触发

            冷却泵保持运行（散热）

            AND 系统自检通过（所有传感器正常，无硬件故障）

  ③ 状态历史记录：

     记录内容：时间戳 + 触发条件 + 当时传感器读数
     供事后故障分析使用

### 10.4.2 降额律验证方法与台架标定

#### 10.4.2.1 为什么需要验证降额律?

设计阶段建立的降额律公式(K₁×K₂×K₃)是基于：
  · 理论估算(换热比例关系)
  · 仿真计算(热网络模型)
  · 供应商数据(电机/泵参数)

但实际系统的行为会与估算有偏差，原因：
  · 污堵系数估算偏差(实际污堵率与假设不符)
  · 环境温度与标准条件差异
  · 元器件参数分散(电机铁芯损耗公差)

**因此必须在台架试验中验证和标定降额律。**

#### 10.4.2.2 验证方法的四个阶段

**阶段1：仿真验证(D60前)**

目标：确认降额律公式的理论正确性

方法：
  □ Simulink+AMESim联合仿真
  □ 构造完整工况矩阵(≥12个点，覆盖K₁/K₂全范围)
  □ 运行仿真至稳态，记录T_winding vs P_avail的关系
  □ 绘制相图(与公式预测的相图叠加对比)

判据：
  ✓ 仿真相图与公式相图的关键点(临界温度)偏差 ≤ ±5°C
  ✓ 绿区/黄区/橙区边界位置偏差 ≤ ±2°C
  ✓ 状态机S0~S6的转移条件都被仿真验证

输出：仿真验证报告(附相图对比图)

**阶段2：台架初步验证(D70~D80)**

目标：在实际硬件上验证降额律的基本趋势

方法：
  □ 台架测试12个工况点(参考表10.4-1)
  □ 每个工况：设定T_sea和Δp，运行至热平衡，记录P_avail
  □ 绘制实测相图，与仿真相图对比
  □ 计算偏差

判据：
  ✓ 实测P_avail vs 仿真P_avail偏差 ≤ ±5%
  ✓ 临界温度T_warn偏差 ≤ ±3°C
  ✓ 无意外的降额"死点"或迟滞

如果偏差>5%：调整仿真模型参数(如污堵系数K₂)，重新标定

输出：台架标定报告(CAL-DERATE-001)

**阶段3：SIT验证(D85~D90)**

目标：在两两集成环节验证状态机逻辑的正确性

方法：
  □ 运行SIT-03(热平衡集成试验，见第11章)
  □ 监测降额状态S0~S6的转移
  □ 验证驾驶台HMI显示与实际状态同步
  □ 触发人工阈值测试(如：加热至T_warn，观察HMI告警)

判据：
  ✓ 状态转移与设计逻辑一致
  ✓ 转移延迟≤5s(允许迟滞，但不能超期)
  ✓ HMI显示准确(绿/黄/橙/红颜色对应正确状态)

输出：SIT-03报告(含状态转移测试记录)

**阶段4：SAT最终验证(D100~D180)**

目标：在实船工况中验证降额律的长期有效性

方法：
  □ 在多个工况点进行海试(沿海巡航、内河巡航等)
  □ 记录T_sea、转速、功率、温度的实时数据
  □ 绘制实船相图，与设计相图叠加
  □ 监测是否出现异常状态转移或边界漂移

判据：
  ✓ SAT相图与设计相图重合(边界偏差≤±2°C)
  ✓ 12个月内相图边界位置无明显漂移(污堵恶化评估)
  ✓ 无意外的停机事件

输出：年度相图验证报告(附实测vs设计对比)

#### 10.4.2.3 降额律标定参数的更新规则

**参数K₁(海水温度因子)**

设计公式：K₁(T_sea) = 1.0 - 0.02×(T_sea - 25)  (T_sea ≥ 25°C)

标定方法(D70~D80)：
  □ 在固定Δp=0(无污堵)的条件下，改变T_sea(20~40°C)
  □ 测量对应的P_avail
  □ 拟合K₁的分段线性或分段二次函数
  □ 若实测数据显示K₁不是线性关系，更新公式

修正示例：
```
设计假设：K₁ = 1.0 - 0.02×(T_sea - 25)  [线性]
实测发现：高温段(T_sea>35°C)偏差较大，应改为二次
修正公式：K₁ = 1.0 - 0.015×(T_sea - 25) - 0.001×(T_sea - 25)²
           (此时35°C时K₁ = 0.880，与实测一致)
```

**参数K₂(污堵因子)**

设计公式：K₂(Δp) = 1.0 - 0.008×(Δp - 20)  (20% ≤ Δp ≤ 50%)

标定方法(D70~D80)：
  □ 固定T_sea(如35°C，最坏情况)
  □ 改变节流阀开度，模拟Δp = 10%, 20%, 30%, 40%, 50%
  □ 测量每个Δp对应的P_avail
  □ 拟合K₂曲线

修正示例：
```
设计假设：K₂ = 1.0 - 0.008×(Δp - 20)  [线性，斜率-0.008]
实测发现：在Δp > 40%时，冷却效率下降更快(非线性)
修正曲线：采用分段函数
  20% ≤ Δp ≤ 40%: K₂ = 1.0 - 0.008×(Δp - 20)  [斜率-0.008]
  40% < Δp ≤ 50%: K₂ = 0.84 - 0.010×(Δp - 40)   [斜率-0.010，更陡]
```

**参数K₃(反馈因子，基于绕组温度)**

设计公式：K₃(T_winding) = MAX(0.3, (T_trip - T_winding)/(T_trip - T_normal))

标定方法(D70~D80)：
  □ 不改变K₃公式(反馈性强，精度通常较好)
  □ 但需要验证T_trip和T_normal两个参数是否准确
  □ 检查约束条件是否合理(如：K₃最小值0.3是否合适)

验证：
  □ 在接近T_trip的温度运行，确认P_avail下降到≤30%(K₃≈0.3对应)
  □ 在T_normal(如130°C)运行，确认P_avail无限制(K₃≈1.0)

### 10.4.3 换热器堵塞突发工况

```
场景描述：
  船舶在热带海域（T_sea=35°C）正常航行（Δp=20%），
  外循环滤器因大量藻类聚集，在 1h 内 Δp 从 20% 快速升至 65%

相图轨迹与控制响应：

  T=0min：(Δp=20%, T_sea=35°C) → P_avail=88%（黄区）
    K₁=0.88, K₂=1.0, K₃=1.0

  T=30min：(Δp=40%, T_sea=35°C) → P_avail=88%×0.84=73.9%（橙区）
    K₂=1-0.008×(40-20)=0.84
    → 触发 L1 报警（P_avail<80%）
    → 驾驶台黄色提示"冷却系统污堵，建议清洗"

  T=50min：(Δp=55%, T_sea=35°C) → P_avail=88%×0.76=66.9%（橙区）
    K₂=0.76（超过50%上限，固定值）
    → 触发橙色报警"换热器污堵严重，已降额至67%，必须清洗"
    → 同时：绕组温度因散热不足开始升高
    → T_winding 从 120°C 缓慢升至 132°C（约 20min 后）
    → 触发 L1 温度报警（T>130°C）→ 冷却泵升至 100%

    K₂=0.76, K₃=1.0（尚未到 155°C 降额阈值）
    P_avail = 88% × 0.76 = 66.9%（控制器已限制）
    绕组温度上升趋势：若不处理，预计 15min 内达到 155°C（L2 降额）

  ✓ 压损监测自动触发两级报警（无需人工干预）
  ✓ 功率自动限制（Δp>50% 时固定 K₂=0.76）
  ✓ 温度趋势预警（上升速率 > 0.5°C/min 时触发趋势告警）
  ✗ 无法自动清洗滤器（需人工操作旁通阀 + 滤芯更换）

### 10.4.4 极端多重故障叠加工况

```
场景描述（压力测试工况，SAT 必须验证）：
  同时发生：
    ① 海水温度突升至 38°C（极端热带海况）
    ② 主冷却泵故障 → 备泵切换（流量降至 70%额定）
    ③ 换热器已有 30% 污堵（Δp=30%）

计算：
  K₁(T_sea=38°C) = 0.88（已达上限，超出设计范围）
  K₂(Δp=30%) = 1 - 0.008×(30-20) = 0.92
  K₃ = 1.0（初始，绕组温度尚正常）
  P_avail_initial = 100% × 0.88 × 0.92 × 1.0 = 80.9%

  备泵切换后，流量 = 70%额定（< 80%额定），
  但尚未触发"流量低"L1 阈值（> 60%），流量降额因子不额外叠加。

  然而换热效率因流量减少而下降（流量 ↓ → ε ↓）：
    ε_actual ≈ ε_0 × (Q/Q_rated)^0.6（湍流传热相关性）
    ε_actual = 0.85 × (0.70)^0.6 ≈ 0.85 × 0.805 = 0.684
    → 冷却能力进一步下降 → 绕组温度缓慢上升

  约 20min 后：T_winding 升至 158°C → 触发 L2 降额（K₃ 介入）：
    K₃ = (170-158)/(170-130) = 12/40 = 0.30
    P_avail = 100% × 0.88 × 0.92 × 0.30 = 24.3%（红区边界！）

控制系统最终响应：
  P_avail < 40% → 进入 S4（严重降额）状态
  保留最低推力（15%额定）维持操纵性
  红色报警 + 要求立即靠港处置
  若 T_winding 继续上升至 175°C → S5 停机

工程启示：
  多重故障叠加（即使每个单独故障均未停机）
  可在 20~30min 内将系统推入停机区
  → 这是设计双泵冗余 + 监控系统"趋势告警"功能的核心依据
  → FMEA 中此场景 RPN 最高，必须优先设计缓解措施
```

### 10.5.1 台架标定流程

```
标定目标：

  ┌──────┬──────────┬──────────┬────────────────────┬────────────┐

  ├──────┼──────────┼──────────┼────────────────────┼────────────┤
  │  1   │ 20       │  0       │ 100%（基准）         │ 验证基准点 │
  │  2   │ 25       │  0       │ 100%                │ K₁起始点   │
  │  3   │ 30       │  0       │ 94.0%               │ K₁线性段   │
  │  4   │ 35       │  0       │ 88.0%               │ K₁设计基准 │
  │  5   │ 38       │  0       │ 88.0%               │ K₁上限验证 │
  │  6   │ 35       │  20      │ 88.0%               │ K₂起始点   │
  │  7   │ 35       │  35      │ 76.2%               │ K₂线性段   │
  │  8   │ 35       │  50      │ 66.9%               │ K₂上限     │
  │  9   │ 35       │  60      │ 66.9%（K₂固定）     │ 超限验证   │
  │  10  │ 30       │  35      │ 71.1%               │ K₁×K₂ 交叉 │

  └──────┴──────────┴──────────┴────────────────────┴────────────┘

每个工况点标定方法：
  ① 设定台架冷却水温（模拟 T_sea）
  ② 设定节流阀开度（模拟 Δp）
  ③ 逐步升高电机功率，至绕组温度达到 130°C（热平衡）
  ④ 记录此时的稳态功率 = P_avail_measured
  ⑤ 计算偏差：error = (P_avail_formula - P_avail_measured) / P_rated × 100%
  ⑥ 若 |error| > 5%：调整对应 K 因子系数

标定结果处理：
  输出：标定报告（CAL-DERATE-001）
        修正后的 K₁/K₂ 分段公式系数

  归档：VCRM 中 REQ-CTRL-007（降额精度）条目关闭证据
  参数更新：修正系数写入控制器 EEPROM

## 10.6 相图在运维中的应用

### 10.6.1 预防性维护决策

```
基于相图的维护决策规则：

  规则1：换热器清洗触发条件
    当工况点在当前 T_sea 下，Δp > 40%（即 K₂ < 0.84）
    → 系统自动推送"换热器清洗建议"（不停机，仅提示）
    → 下次靠港（≤72h内）安排清洗
    
    当 Δp > 50%（K₂ = 0.76）
    → 驾驶台橙色告警"换热器必须清洗"
    → 尽快靠港，不建议延误超过 24h
    
    清洗后操作：
    → 控制器维护菜单执行"压损基准值重置"
    → 系统记录清洗日期和 Δp 初始值（维护档案）

  规则2：滤器清洗触发条件
    外循环：Δp 升高速率 > 5%/h（斜率告警）
    → 提示"外循环滤器可能堵塞，检查并清洗"
    
    内循环：若流量异常（< 90%额定 且 泵转速正常）
    → 提示"内循环滤器可能堵塞"

  规则3：航线规划建议
    当预报前方海域 T_sea > 35°C（>3h 持续），
    且当前 Δp > 30%（K₁ × K₂ < 0.75）：
    → 系统建议：靠港清洗换热器后再通过热带海域
    → 或：提前以 80%额定功率航行（降低热负荷裕度消耗）

  规则4：绩效退化趋势分析
    每月统计相同工况点（相同 T_sea，相同功率）下的
    实际绕组温度趋势：
      若绕组温度持续升高（月均 +2°C）：
      → 可能原因：换热器逐渐结垢（未完全清洗）
                 绕组绝缘老化（绝缘电阻年检）
                 轴承热阻增大（定期振动检查）
    → 触发计划性大修评估
```

### 10.6.2 能效优化应用

```
基于相图的能效优化（超越"降额保护"的主动能效管理）：

  最优工作点选择：
    在满足推力需求的前提下，
    选择效率最高的（转速，转矩）组合点
    该最优点应在相图绿区深处（远离边界），
    且在电机效率 MAP（见第11章 FAT 试验）高效岛内
    
    示例：
      目标推力：60%额定推力
      可实现方式A：n=80%额定转速，T=75%额定转矩
                   → 效率 93%，T_winding=118°C（绿区）
      可实现方式B：n=90%额定转速，T=67%额定转矩
                   → 效率 91%，T_winding=110°C（绿区）
      → 选择方式A（效率更高，仍在绿区安全范围）

  冷却泵能耗优化：
    传统做法：冷却泵始终运行在 100%转速（简单可靠，但浪费）
    优化做法：根据相图当前位置动态调节泵转速
      绿区深处（P=100%，T_winding<100°C）：泵转速降至 70%
      黄区（T_winding=100~130°C）：泵转速升至 85%
      橙区（L2 降额）：泵转速升至 100%
      
      节能效果：
        泵功率 ∝ n³，转速从 100% 降至 70% → 功率降至 34%
        若冷却泵额定功率 3kW，正常运行在 70% 转速 → 节省 ~2kW
        年节电：2kW × 6000h（年运行）= 12000 kWh ≈ 节省 1.5 万元

  余热利用接口（预留）：
    相图橙区运行时（冷却液出口温度 > 60°C），
    冷却液高温段可通过板换为船舱采暖提供热源
    → 降低电热采暖能耗（北方内河冬季航线有效）
    → 接口预留在 ICD-CS-004（船总体–冷却系统接口）
```

## 10.7 本章小结与关键输出物

### 10.7.1 核心要点回顾

```
本章核心要点（六条）：

  ① 相图是"系统能力地图"，不是"事后分析工具"
     实时显示于驾驶台，使船员在任意时刻都知道
     "当前系统还剩多少余量"，而非等到报警才响应

  ② 降额律三因子（K₁×K₂×K₃）分别应对三类退化：
     K₁：外部热汇温度升高（无法控制，只能降额）
     K₂：换热器污堵（可通过清洗恢复，属于可逆退化）
     K₃：绕组温度实测反馈（前馈失效时的最后兜底）

  ③ 降级走廊的核心价值是"主动降级 vs 被动停机"
     7级状态机（S0~S6）确保系统在任意工况下
     都有有序的退出路径，永远保留最低推力（15%额定）
     使船只维持最低操纵性，安全靠港

  ④ 迟滞设计是防止控制振荡的关键
     恢复阈值 ≠ 触发阈值，差值为 5~10°C + 时间延迟 30~300s
     防止温度在阈值附近波动时反复触发降额/恢复切换

  ⑤ 标定是相图可信度的保证
     K₁/K₂ 系数必须通过台架试验（12个工况点）标定
     未标定的理论公式预测误差可达 ±15%，远超 ±5% 要求
     标定结果写入控制器 EEPROM，随系统全寿命保存

  ⑥ 相图不仅用于保护，更用于运维决策和能效优化
     换热器清洗时机、航线规划、冷却泵节能调速，
     均可基于相图当前位置和趋势做出主动决策
     从"被动保护"进化为"主动能效管理"
```
