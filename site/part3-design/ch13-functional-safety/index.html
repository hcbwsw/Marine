    第13章 功能安全工程¶

    IEC 61508/61511 体系与船用电驱推进系统¶

    本章导读

    功能安全（Functional Safety）是新能源电推船的硬性要求，也是船级社审图的必问项目。

    本章回答三个核心工程问题： - ① 推进系统哪些失效是"安全关键"的？需要什么等级的保护？（安全功能识别与SIL定级） - ② 这些安全功能的架构如何设计，才能确保达成SIL要求？（单/双/三通道权衡） - ③ 如何通过计算和验证证明设计满足了SIL要求？（PFD/PFHR计算、FAT/SAT验证）

    船用电推系统的功能安全与通用IEC 61508的主要区别： - 船舶运营环境的自主性高（海上无外援，故障代价大） - 推进失效直接导致船舶失控（安全临界性强） - 规范强制要求：E-STOP必须硬接线，第二失效防线必须独立

    本章关键输出物： - 安全功能清单（6项：SF-01~SF-06）+ SIL定级矩阵 - E-STOP硬接线标准设计图纸（双通道，SIL2） - PFD/PFHR计算表格（含诊断覆盖度、共因失效系数） - 安全功能架构设计文档（单/双/三通道对标） - SIL认证文件清单与第三方评估流程

    17.1 IEC 61508 标准体系与船舶规范的对标¶

    17.1.1 标准框架层级关系¶

    国际通用标准：
  IEC 61508（功能安全与可编程控制系统）
    ├─ Part 1：概念与定义 ← SIL定义、失效分类、危害分析
    ├─ Part 2：硬件可靠性 ← 故障率数据库、冗余架构原则
    ├─ Part 3：软件设计 ← V模型、代码走查、测试覆盖率
    ├─ Part 4：定义表例 ← 工程示例、计算工具
    └─ Part 7：IEC 61800应用 ← 变频器/电机驱动器特定要求

产业特定标准：
  IEC 61511（过程工业的功能安全）← 基于61508，针对装置类系统
  IEC 61800-5-1（驱动器/变频器的安全功能） ← 电力电子专用
  ISO 13849-1（机械系统的功能安全） ← 传统机械驱动的安全

船舶规范：
  CCS《钢质海船入级规范 第4篇 第12章 电力推进系统》
    ├─ 强制要求：安全停机回路硬接线独立实现
    ├─ 强制要求：主给电和应急推进间隔离
    ├─ 参考要求：FMEA分析，安全功能SIL评估
    └─ 推荐要求：冗余配置（双泵/双变频器）

  DNV GL《Part 4 Chapter 8 Electrical Power Systems》
    ├─ 明确强制：安全功能的SIL等级定义与验证
    ├─ 明确强制：FMEA报告 + PFD计算 + 架构设计文档
    ├─ 强制要求：E-STOP应急停机响应时间≤10s
    └─ 推荐要求：第三方SIL认证（TÜV/DNV等）

    17.1.2 SIL定义与船用推进系统的对标¶

    SIL等级 目标失效概率 PFD/年 平均故障间隔 MTTF 含义 船用应用场景
    SIL0 >10⁻¹ <10年 无功能安全要求 电源指示灯、显示屏显示
    SIL1 10⁻¹~10⁻² 10~100年 低完整性级 基础停机保护（如热保护限流）、启动报警
    SIL2 10⁻²~10⁻³ 100~1000年 中完整性级 应急停机(E-STOP)、直流高压放电、编码器冗余
    SIL3 10⁻³~10⁻⁴ >1000年 高完整性级 双推进器协调控制、多路冗余系统
    SIL4 <10⁻⁴ >10000年 最高完整性级 不推荐用于船舶（成本极高，验证困难）

    对标说明： - SIL2相当于系统在1年运营中失效概率≤1%（故障仍然会发生，但足够稀少） - SIL3相当于MTTF>1000年，即某功能故障须在船舶服役期(20~25年)内不超过2次 - 船用E-STOP通常设计为SIL2（双通道冗余架构），不设计SIL3（成本-收益不经济）

    17.2 船用电驱系统六大安全功能识别与SIL定级¶

    17.2.1 六大安全功能表¶

    SF-01：应急停机(E-STOP) → SIL2¶

    项目 规格
    失效模式 驾驶员按E-STOP，推进器不停
    失效严重度S 4（船舶失控风险）
    触发频率O 2（应急频率低）
    可控性Ct 2（操作员有5~10s反应时间）
    建议SIL 2（双通道硬接线）
    设计方案 两个独立继电器，各自直接驱动变频器STO信号
    验证方法 HIL故障注入 + 现场测试（按钮按下→推进力立即降为0）

    SF-02：热保护停机 → SIL2（升级）/ SIL1（基础）¶

    项目 规格
    失效模式 电机温度超140°C，系统不停机
    失效严重度S 4（火灾风险）
    触发频率O 2（工作在热极限的频率）
    可控性Ct 3（运行中难以察觉）
    建议SIL 2（三传感器投票升级至SIL2）
    设计方案 3×PT100投票，当2个传感器超阈值即停机；单个传感器故障报警但继续运行（降级）
    验证方法 台架温升试验 + 传感器故障注入

    SF-03：直流母线快速放电 → SIL2¶

    项目 规格
    失效模式 断电后母线电压未降至<60V（触电风险）
    失效严重度S 3（触电风险）
    触发频率O 3（每次停机触发）
    可控性Ct 3（隐性风险，难以察觉）
    建议SIL 2（主动+被动双重放电）
    设计方案 主动制动电阻(8Ω,脉冲) + 备用泄放电阻（被动）
    验证方法 实测放电时间≤1s（示波器测量）

    SF-04：轴承过温保护 → SIL1¶

    项目 规格
    失效模式 推力轴承温度>75°C，系统无降额响应
    失效严重度S 3（轴承加速损坏）
    触发频率O 2（海上运行工况低频）
    可控性Ct 3（隐性损伤）
    建议SIL 1（单传感器软件保护+硬件温度开关备份）
    设计方案 PT100软件监测，>75°C进入S4降级状态；>80°C硬件开关强制停机
    验证方法 台架加热试验，读取传感器和硬件开关反应时间

    SF-05：失推保护(Lost Thrust) → SIL2¶

    项目 规格
    失效模式 变频器故障导致推力突然消失
    失效严重度S 4（船舶失控）
    触发频率O 1（极低频）
    可控性Ct 3（突然断推）
    建议SIL 2（硬接线+软件双重监测）
    设计方案 主动停机继电器（控制器故障时通过硬接线强制停机）+ 编码器故障检测（软件反馈）
    验证方法 HIL注入变频器故障，验证停机逻辑触发；FAT现场测试

    SF-06：通信超时保护(Watchdog) → SIL1¶

    项目 规格
    失效模式 CAN/通信中断，系统无响应继续运行
    失效严重度S 2（功能丧失，但不立即危险）
    触发频率O 1（极低频）
    可控性Ct 1（驾驶员可发现）
    建议SIL 1（软件看门狗WDT）
    设计方案 CAN超时2s触发WDT中断，立即停机；LED报警"通信故障"
    验证方法 HIL场景09：通信中断，验证停机响应≤3s

    17.3 HAZOP方法推导安全功能¶

    故障链条分析（以热失控为例）：

传感器失效 → 无温度读数 → 控制器无法感知 → 无限功率 
  → 绕组过热 → 绝缘击穿 → 火灾

对应的安全功能需求：
  ① 传感器冗余（3个）
  ② 传感器失效检测逻辑（三传感器投票）
  ③ 故障时保守降额（假设最坏情况，直接停机）
  ④ 硬接线温度开关（第二道防线，完全独立）

SIL等级确定：
  顶事件严重度S=4（火灾）
  发生频率O=2（设计工况下频率低）
  可控性Ct=3（运行中难以被船员察觉）
  → 按矩阵得SIL=2（单点故障不导致完全失效）

    17.3.1 SIL定级矩阵（ASIL标准方法改编）¶

│O3 │          │          │          │          │

│O2 │          │          │          │          │
├───┼──────────┼──────────┼──────────┼──────────┤

    17.4 双通道硬接线E-STOP标准设计¶

    17.4.1 硬接线原理¶

    为什么E-STOP必须硬接线？

    场景：海浪打在船舱，驾驶员按E-STOP按钮

❌ 软件保护弱点：
   按钮→中断→MCU处理→驱动器指令停机
   问题：如果MCU已经瘫痪呢？中断无法响应！

✅ 硬接线强点：
   按钮→继电器线圈通电状态下维持触点闭合
   按钮按下→继电器线圈断电→触点弹开
   E-STOP线路：直接驾驶驱动器的STO(Safe Torque Off)信号
   100%物理级别强制，绕过所有软件

    17.4.2 标准双通道E-STOP设计图¶

    ┌─────────────────────────────────────────────────────────┐
│ E-STOP 硬接线标准拓扑图（双通道，SIL2）                 │
├─────────────────────────────────────────────────────────┤

驾驶台E-STOP按钮（24VDC）
    │
    └──┬─ 通道A ─ [继电器1] ─→ [触点A] ─→ 变频器1-STO
       │
       └─ 通道B ─ [继电器2] ─→ [触点B] ─→ 变频器2-STO

机舱E-STOP按钮（备用，独立供电）
    │
    └──┬─ 通道C ─ [继电器3] ─→ [触点C] ─→ 变频器1-STO
       │
       └─ 通道D ─ [继电器4] ─→ [触点D] ─→ 变频器2-STO

关键特性：
  ① 两个按钮（驾驶台+机舱），独立供电
  ② 每个按钮驱动两个继电器（双冗余）
  ③ 每个继电器的触点独立驱动一个变频器
  ④ 触点必须是"常开-按下闭合"设计
  ⑤ 断电时触点自动弹开（fail-safe原理）

    17.4.3 电气规格与验收准则¶

    参数 规格值 理由
    线圈电源 24VDC（独立不依赖推进系统电源） 推进系统失电时仍可工作
    线圈继电器 通用工业级（不低于2A@24V） 可靠工作点裕度
    触点材质 银镉合金/银镍合金 耐弧光烧蚀
    触点规格 ≥2个常开触点，额定≥1A@24V 每个触点独立驱动一个变频器STO
    响应时间 ≤100ms（从按下到推进力为0） 安全停机速度要求
    检测周期 年1次外观检查+功能测试 船舶定期检验
    隔离要求 E-STOP线与动力线、控制线分开敷设 EMC隔离

    17.5 PFD 与 PFHR 的计算方法¶

    17.5.1 PFD 公式与计算¶

    公式(按需运行模式)： 

    PFD = Σ(λ_i × T_i / 2)

其中：
  λ_i = 第i个部件的失效率 (failures/year)
  T_i = 相邻两次诊断的时间间隔

    船用E-STOP双通道PFD计算示例：

    系统架构：
  继电器K1（线圈）→ 触点1驱动变频器1-STO
  继电器K2（线圈）→ 触点2驱动变频器2-STO
  工作原理：STO线路"或"关系（任一触点打开即停机）

故障率数据（IEC 61508参考值）：
  λ_coil  = 0.001 /年    (继电器线圈开路)
  λ_contact = 0.005 /年  (触点粘连/接触不良)
  诊断间隔 T = 1年（每年定检一次）

单通道PFD计算：
  PFD_single = (λ_coil + λ_contact) × T / 2
             = (0.001 + 0.005) × 1 / 2
             = 0.003 = 0.3%

双通道PFD（两个继电器）：
  共因失效系数 β_CCF = 0.05（表示5%故障是共同原因）

  PFD_dual ≈ PFD_single × (1 - β_CCF) × 0.1
           = 0.003 × 0.95 × 0.1
           = 0.000285 ≈ 0.03%

目标对标：
  SIL2要求：PFD < 1% ✅ 满足
  SIL3要求：PFD < 0.1% ⚠️ 接近（需要三通道）

    17.5.2 诊断覆盖度(DC)与故障率修正¶

    诊断覆盖度定义：系统能否检测出该失效模式的能力

E-STOP系统的DC评估：
  失效模式              |  检测能力  | DC% 
  ─────────────────────┼─────────────┼────
  继电器线圈开路         | 可检测     | 90%
  （通过STO反馈检查）    |            |
  继电器触点粘连         | 难检测     | 30%
  （除非主动测试）       |            |
  触点接触不良           | 中等检测   | 60%
  （通过偶极性检查）     |            |

修正公式：
  λ_detected = λ_total × DC
  λ_undetected = λ_total × (1 - DC)

PFD修正后：
  PFD_corrected = λ_undetected × T / 2 
                + λ_detected × λ_diagnostic_failure

结论：高可靠性需要定期主动测试（月度功能检查）

    17.6 SIL 认证的第三方评估流程¶

    17.6.1 认证三个阶段¶

    Stage 1：产品研发与内部评估（供应商责任，3-6个月）

    工作内容：
  ① FMEA分析（系统级/子系统级）
  ② 功能安全计划编制
  ③ 安全架构设计（SIL架构确定）
  ④ PFD/PFHR计算
  ⑤ 危险分析(HAZOP/LOPA)

输出物：
  □ 功能安全计划(FSP) - 设计阶段
  □ FMEA报告 - 失效模式分析
  □ 架构设计文档 - SIL实现方案
  □ 计算书 - PFD定量评估

质量保证：
  内部技术评审 ≥ 2次
  与船级社预沟通 ≥ 1次

    Stage 2：设计评审与第三方验证（6-12个月）

    船级社角色：
  ① 审查安全计划是否符合规范
  ② 审查FMEA的完整性
  ③ 审查PFD计算的合理性
  ④ 确认SIL架构的可行性

典型问题：
  Q1：PFD计算中的故障率数据来源不明
  A1：所有数据来自IEC 61508 Part 2附录A

  Q2：E-STOP响应时间如何验证？
  A2：见FAT计划中的测试项TE-07

  Q3：共因失效的β系数如何确定？
  A3：两个继电器来自不同厂商，β_CCF=0.05（保守）

    Stage 3：建造中检验与FAT验证

    FAT关键项（船级社见证）：

□ E-STOP硬接线功能测试
  - 驾驶台按钮：推进力→0，响应时间≤100ms
  - 机舱按钮：同上
  - 按钮锁定确认：按钮弹回自动复位
  - 冷启动测试：上电后立即按E-STOP

□ 传感器冗余验证(SF-02热保护)
  - 单传感器故障：系统报警但不停机（降额至安全等级）
  - 双传感器故障：系统自动停机
  - 三传感器全部故障：强制停机（fail-safe）

□ 故障诊断覆盖测试
  - CAN通信中断：推进器停止（WDT超时）
  - 编码器信号断裂：系统报警并限速
  - 继电器被逐个隔离（打开触点）：验证独立性

试验报告格式：
  ├─ 试验目的
  ├─ 试验条件（环境/参数）
  ├─ 测试项与结果
  ├─ 合格判定
  ├─ 签字盖章（供应商+船级社）
  └─ 附件（数据表/示波器波形）

    17.7 常见的 SIL 认证失败原因与预防¶

    失败原因 症状 预防措施
    故障率数据不可溯源 审图返工说"λ值从哪来的？" 所有数据标注来源（如IEC 61508 Part 2 Table D.1）
    PFD计算遗漏共因失效 审查说"两个继电器可能同时失效" 明确说明β_CCF参数与选择理由（不同厂商/不同环境等）
    诊断覆盖度评估过乐观 审查说"这个故障怎么检测？" DC评估时保守：可检测=90%，难检测≤30%
    E-STOP响应时间超限 实测120ms，规范要求≤100ms 增加并联触点或升级继电器型号
    缺少失效模式验证 说有故障检测，但没试过 FAT必须有"注入故障+验证系统反应"的测试项
    安全功能间互相干扰 SF-02热保护与SF-01E-STOP冲突 设计评审时明确各安全功能的优先级与隔离机制
