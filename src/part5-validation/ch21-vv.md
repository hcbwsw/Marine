# 第20章 验证与确认体系（V&V）

> **本章导读**
> "没有被验证的设计等于没有设计"——
> V&V（Verification & Validation）体系回答两个问题：
>
> - **Verification（验证）**：我们是否把产品做"对"了？
>   （设计输出是否满足设计输入/需求规格？）
> - **Validation（确认）**：我们是否做了"对的"产品？
>   （整个系统是否满足船东/任务的真实需求？）
>
> 船用电推V&V体系的特点：
> ① **分层递进**——组件FAT→子系统台架→全系统SIT→航行试验SAT
> ② **VCRM闭环**——每条需求都有对应验证证据，不漏项
> ③ **仿真先行**——HIL验证在台架试验前完成，降低实物试验风险
> ④ **规范强制**——部分验证项目由船级社规范强制规定，不可裁减

## 14.1 验证方法分类与选用原则

### 14.1.1 四类验证方法定义

```
按 INCOSE SE Handbook 及 ISO 15288 定义，验证方法分四类：

  ┌──────────┬──────────────────────────────┬──────────────┬───────────┐
  │ 验证方法  │ 定义                          │ 适用场景      │ 成本等级  │
  ├──────────┼──────────────────────────────┼──────────────┼───────────┤
  │ 分析(A)   │ 用数学/理论计算证明需求被满足  │ 早期设计阶段  │ 低        │
  │ 仿真(S)   │ 用计算机模型验证性能/行为      │ 中期详细设计  │ 低~中     │
  │ 检查(I)   │ 目视/文件检查确认符合规范      │ 设计/制造完成 │ 低        │
  │ 试验(T)   │ 实物测试获取验证数据           │ 原型/产品阶段 │ 高        │
  └──────────┴──────────────────────────────┴──────────────┴───────────┘

  选用原则（优先级）：
    能用分析/仿真解决的，不做实物试验（节省成本）
    但以下情形必须做试验（不可用仿真替代）：
      ① 船级社规范明确要求型式试验的项目
      ② 新技术/新材料（无历史数据支撑仿真可信度）
      ③ 安全关键功能（热失控传播、紧急停机响应时间）
      ④ 仿真模型未经试验标定的场景

  验证策略决策矩阵：
  ┌──────────────────────┬──────┬──────┬──────┬──────┐
  │ 需求类型              │  A   │  S   │  I   │  T   │
  ├──────────────────────┼──────┼──────┼──────┼──────┤
  │ 性能指标（效率/推力）  │  ✓   │  ✓   │  —   │  ✓*  │
  │ 热管理（温升）        │  ✓   │  ✓   │  —   │  ✓*  │
  │ 安全保护逻辑          │  —   │  ✓   │  ✓   │  ✓*  │
  │ 机械强度/寿命         │  ✓   │  ✓   │  —   │  ✓   │
  │ 电磁兼容（EMC）       │  —   │  —   │  —   │  ✓*  │
  │ 噪声/振动             │  ✓   │  ✓   │  —   │  ✓*  │
  │ 环境试验（盐雾/低温）  │  —   │  —   │  —   │  ✓*  │
  │ 软件功能逻辑          │  —   │  ✓   │  ✓   │  ✓   │
  │ 接口兼容性            │  —   │  ✓   │  ✓   │  ✓   │
  └──────────────────────┴──────┴──────┴──────┴──────┘
  注：✓* = 船级社审图强制要求必须做试验的项目
```

### 14.1.2 VCRM——验证交叉参考矩阵

```
VCRM（Verification Cross-Reference Matrix）是V&V体系的核心文件：

  标准VCRM行结构（每条需求一行）：
  ┌──────────┬───────────────────────┬──────┬──────────┬──────────┬────────┬────────┐
  │ 需求编号  │ 需求描述               │ 来源  │ 验证方法  │ 验证阶段  │ 验证状态│ 证据编号│
  ├──────────┼───────────────────────┼──────┼──────────┼──────────┼────────┼────────┤
  │REQ-SYS-001│绕组温升≤80K（连续额定）│CCS规范│ 试验(T)  │ SIT台架  │ 关闭   │RPT-001 │
  │REQ-COOL-001│冷却液入口温度≤45°C   │系统需求│分析+试验 │ FAT+SIT  │ 关闭   │RPT-002 │
  │REQ-BAT-003│热失控传播测试通过     │CCS规范│ 试验(T)  │ 型式试验  │ 关闭   │CERT-001│
  │REQ-CTRL-007│螺旋桨出水保护响应≤2s │船东需求│ 仿真+试验│ HIL+SIT  │ 进行中 │ —      │
  └──────────┴───────────────────────┴──────┴──────────┴──────────┴────────┴────────┘

  VCRM管理原则：
    ① 每条需求必须有且仅有一个主验证方法
    ② 验证状态：开放(Open) → 进行中(In Progress) → 关闭(Closed)
    ③ 关闭条件：验证证据（报告/证书/数据）完整且符合验收准则
    ④ 项目里程碑审查：VCRM关闭率 = 关闭条目数 / 总条目数 × 100%
       D0（需求冻结）：VCRM建立，关闭率0%
       M3（子系统设计完成）：A类/I类验证关闭率≥80%
       M5（SIT完成）：关闭率≥95%
       M6（SAT完成）：关闭率100%（所有条目必须关闭，方可交船）
    
  最小VCRM条目数（典型500kW内河纯电推进系统）：
    系统级需求：约25条
    冷却子系统：约20条
    传动子系统：约25条
    控制子系统：约30条
    新能源特有（第19章）：约15条
    合计：约115条（作为下限参考）
```

## 14.3 HIL验证——控制软件的主战场

### 14.3.1 HIL场景完整清单（扩展版）

```
船用电推HIL必测场景（最小集合，共18个场景）：

  ━━ 基础功能验证（场景1~5）━━
  
  场景01：正常启动-加速-巡航-减速-停机
    注入：驾驶台转速指令阶跃序列（0→50%→100%→50%→0）
    验证：转速跟踪精度≤±0.5%，无过冲>5%
          启动电流≤150%额定（保护限流有效）
          停机后直流母线放电至<60V时间≤1s
  
  场景02：倒车操纵（全速前进→全速倒车）
    注入：转速指令阶跃 +100% → -100%（最恶劣场景）
    验证：直流母线电压峰值≤1.15×额定（制动电阻吸收有效）
          倒车推力在5s内达到额定倒车值
          轴系扭矩峰值≤联轴器额定峰值扭矩
  
  场景03：最大持续功率运行（2h热稳态）
    注入：100%功率指令，注入T_sea=35°C（最热设计工况）
    验证：2h后绕组温度稳定≤T_warn（130°C）
          冷却液出口温度≤65°C
          泵转速调节曲线合理（无频繁振荡）
  
  场景04：低速精确操纵（靠泊工况）
    注入：转速指令在5%~20%额定之间阶跃，模拟靠泊调车
    验证：低速转矩控制精度≤±5%
          最低稳定转速≤5%额定（内河靠泊需求）
          无速度振荡（速度纹波≤±1%）
  
  场景05：禁速区快速穿越（扭振临界转速避让）
    注入：转速斜坡指令经过扭振禁速区（如40~50Hz对应转速段）
    验证：穿越时间≤10s（不在禁速区内停留）
          穿越时扭矩波动≤±10%
          禁速区标志位在控制器日志中正确记录

  ━━ 热管理与保护（场景6~9）━━
  
  场景06：绕组温度三级保护验证
    注入：温度传感器信号依次注入130°C→155°C→175°C
          每级保持持续3min
    验证：
      130°C：冷却泵升至100%，黄色报警，功率不降额
      155°C：线性降额开始（功率降至目标值±5%）
      175°C：停机指令发出时间≤0.5s
             停机后冷却泵继续运行（散热）
  
  场景07：冷却液流量低保护
    注入：流量传感器信号从100%→60%→30%（阶跃）
    验证：
      <60%额定：10s内触发降额至70%功率
      <30%额定：2s内触发停机
      单泵失效：备泵自动启动≤5s，功率自动恢复
  
  场景08：换热器污堵模拟（压差增大）
    注入：外循环压差信号逐步增加至150%初始值，再到200%
    验证：
      >150%：触发"换热器堵塞"报警（黄色）
      >200%：触发降额至80%功率（橙色报警）
      降额后温升重新稳定（验证降额量足够）
  
  场景09：海水温度阶跃（极端热天气）
    注入：T_sea从25°C在10min内线性升至40°C
    验证：
      降额律K₁自动调整（功率从100%下降至约80%）
      绕组温度在降额后稳定在安全区间
      驾驶台显示"高温海水降额运行"提示

  ━━ 故障注入（场景10~14）━━
  
  场景10：螺旋桨出水保护（海船专用）
    注入：负载转矩信号周期性阶跃（满载→5%→满载，周期8s）
          模拟Hs=3m海浪下螺旋桨出入水
    验证：
      转矩<15%额定持续>200ms：限速保护激活（转速限至85%额定）
      重新入水：转矩恢复速率≤30%额定/s
      无"飞车"现象（转速不超过110%额定）
  
  场景11：编码器故障与无速度传感器切换
    注入：编码器信号中断（持续10ms，然后长期中断）
    验证：
      中断10ms：控制器切换至无速度传感器模式（MTPA估算）
      速度估算误差≤±5%（对比实际转速）
      5s内驾驶台报警"编码器故障，降级运行"
  
  场景12：主控制器失电后的紧急操作
    注入：Level 2控制器突然断电（100ms后恢复）
    验证：
      断电期间：变频器转入安全停机（STO模式）
      恢复供电后：控制器热重启完成≤5s
      重启后：等待驾驶台确认指令后才能重新加速
  
  场景13：传感器信号漂移（温度传感器偏置注入）
    注入：单个绕组温度传感器信号偏置+20°C（其余正常）
    验证：
      三传感器投票机制：偏差传感器被识别为异常
      采用其余两路均值，不触发误停机
      驾驶台报警"温度传感器异常，建议检查"
  
  场景14：直流母线绝缘故障模拟
    注入：IMD绝缘电阻信号：500kΩ→100kΩ→20kΩ（依次降低）
    验证：
      <500kΩ：驾驶台黄色预警"绝缘监测异常"
      <100kΩ：橙色报警"绝缘电阻偏低，请检查"
      <20kΩ：红色报警"绝缘严重故障"，降额并限制充电

  ━━ 新能源特有场景（场景15~18）━━
  
  场景15：热失控联动响应（BMS→推进控制器）
    注入：BMS热失控信号TR3（电池舱CO>50ppm触发）
    验证：
      推进控制器收到TR3信号≤200ms内降额至60%
      充电接触器断开≤500ms
      驾驶台红色报警"电池异常，准备应急"
      灭火系统请求信号输出（继电器动作确认）
  
  场景16：低温充电限制验证
    注入：电池温度信号分别注入：-10°C / 0°C / 5°C / 10°C
          同时模拟充电桩连接
    验证：
      -10°C：充电请求被拒绝（接触器不闭合）
      0°C：充电电流限制至0.05C（验证≤实际0.05C）
      5°C：充电电流限制至0.1C
      10°C：充电电流限制至0.3C
      各温度切换时过渡平滑（无冲击）
  
  场景17：SOC耗尽应急处置
    注入：SOC信号从20%线性降至5%
    验证：
      SOC=15%：驾驶台黄色"电量低，建议靠港"
      SOC=10%：推进功率自动降额至50%（节能续航模式）
      SOC=5%：推进功率降额至20%（应急低速模式）
              驾驶台强制报警（声光持续）
      控制器禁止任何快充指令干扰（此时保护续航为最高优先级）
  
  场景18：多机并联故障切换（双轴推进，一机失效）
    注入：左推进电机故障停机信号
    验证：
      右推进电机自动提升功率至100%（补偿推力）
      艏侧推收到偏航修正指令（若有艏侧推）
      驾驶台报警"左推进故障，单机推进模式"
      单机模式下转速控制仍稳定（无振荡）
```

## 14.7 本章小结与关键输出物

```
V&V体系核心要点（五条）：

  ① VCRM是V&V体系的骨架
     每条需求一行，不漏项，关闭率=100%才能交船
     典型系统约115条需求，建立之初即须完整，不可事后补充

  ② 验证方法四类，试验最贵但不可替代
     安全关键功能（热失控/急停/放电）必须实物试验
     日常性能指标可组合使用分析+仿真+试验（降低成本）

  ③ HIL是控制软件验证的主战场
     18个场景覆盖基础功能/热管理保护/故障注入/新能源特有
     HIL通过是SIT台架试验的前提条件（减少实物试验风险）

  ④ FAT是质量门控的第一关
```
