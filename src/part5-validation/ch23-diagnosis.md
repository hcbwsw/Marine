## 18.1 故障码定义与分类体系

### 18.1.1 故障码命名规则（五维体系）

```
故障码命名规则：[系统]-[子系统]-[故障类型]-[序号]
例如：CL-TH-OVT-001  
     └─冷却 └─热传输 └─过温 └─#001

分类维度1：系统级（3个字母）
  CL = Cooling（冷却系统）
  DR = Drive（驱动-电机+传动）
  CT = Control（控制系统）
  PW = Power（电源系统）
  NW = Network（网络/通信）

分类维度2：子系统级（2个字母）
  冷却系统：
    TH = Thermal（热交换）
    PM = Pump（泵）
    SN = Sensor（传感器）
    VL = Valve（阀门）
  
  驱动系统：
    MT = Motor（电机）
    BG = Bearing（轴承）
    SH = Shaft（轴系）
    CP = Coupling（联轴器）
    PR = Propeller（螺旋桨）
  
  控制系统：
    MCU = Microcontroller（控制器）
    CD = Encoder（编码器）
    CAN = CAN-BUS（通信）
    HMI = Human-Machine Interface（驾驶台）

分类维度3：故障类型（4个字母缩写）
  OVT = Over-Temperature（过温）
  LFT = Low Flow Temperature（低流量）
  HIGH_PRES = High Pressure（高压）
  CAN_TIMEOUT = CAN通信超时
  SENSOR_FAULT = 传感器故障
  HW_FAULT = Hardware Fault（硬件故障）

分类维度4：优先级/动作
  E = Emergency(紧急，≤5s内停机)
  W = Warning(警告，可继续有限运行)
  I = Information(信息，无需立即处理)
```

## 18.2 故障诊断决策树设计

### 18.2.1 故障隔离三层级

```
故障隔离的三个层级：

L1：系统级隔离（是哪个子系统出问题？）
    ├─ 冷却系统  → 检查流量/温度/压力三个参数
    ├─ 驱动系统  → 检查电机温度/振动/负载电流
    ├─ 控制系统  → 检查通信/传感器信号/HMI反馈
    ├─ 电源系统  → 检查直流母线/交流输入/风扇
    └─ 通信系统  → 检查CAN/以太网/远程连接

L2：子系统内隔离（具体是哪个部件？）
    冷却系统内：
    ├─ 泵故障  → 流量=0，压力=0
    ├─ 管路堵塞 → 流量<50%，压力升高
    ├─ 换热器污堵 → 流量降低，温度升高
    └─ 传感器故障 → 三传感器读数互相矛盾

L3：故障根本原因分析（为什么失效？）
    泵故障中：
    ├─ 轴密封泄漏 → 泵转但无压力，液位下降
    ├─ 叶轮堵塞  → 泵转但流量低，功率高
    ├─ 轴承坏   → 泵不转，异响
    └─ 电源故障 → 泵完全不动，电流=0
```

### 18.2.3 典型故障与诊断特征表

| 故障模式 | 温度 | 流量 | 压力 | 电流 | 诊断特征 | 根本原因 |
|---------|-----|-----|-----|-----|---------|--------|
| 泵轴密封泄漏 | ↑ | ↓ | ↓ | ↓ | 转速正常，压力低，流量低，液位下降 | 密封磨损 |
| 换热器污堵 | ↑ | ↓↓ | ↑↑ | ↑ | 转速正常，压力高，流量低，温度无法控制 | 冷却液污染 |
| 管路堵塞 | ↑ | ↓ | ↑ | ↑ | 转速上升（负载下降），压力升高，温度上升 | 杂质积累 |
| 泵轴承坏 | 可能↑ | 0 | 0 | 0 | 泵不转，可能有异响，液位下降 | 轴承疲劳 |
| 控制阀卡死 | ↑ | ↓ | ↑ | 正常 | 流量低，压力高，温度上升，阀位置不变 | 内部积碳 |
| 传感器漂移 | 读数↑ | 正常 | 正常 | 正常 | 单传感器读数异常，其他正常 | 传感器老化 |
| 温度开关故障 | 实际110°C | 正常 | 正常 | 正常 | 系统报警不停机，手动测温验证 | 开关卡死 |

### 18.3.1 冷却系统故障排除检查单

```
╔════════════════════════════════════════════════╗
║  冷却系统故障排除检查单                         ║
║  (可贴在机舱，配合语音播报使用)                ║
╚════════════════════════════════════════════════╝

【症状】绕组温度报警（CL-TH-OVT-001，黄灯）

┌─ Step 1：观察流量表 ────────────────────────┐
│ ☐ 流量正常(>70%Qrated)?                      │
│   └─ YES → Go to Step 2                       │
│   └─ NO  → Go to Step A（流量故障分支）      │
└────────────────────────────────────────────┘

┌─ Step 2：检查温度传感器 ────────────────────┐
│ ☐ 驾驶台显示三个温度读数?                    │
│   └─ 全部显示  → 检查数值一致性                │
│   └─ 缺失      → 传感器故障，报告技术员       │
│                                              │
│ ☐ 三个温度值偏差<5°C?                       │
│   └─ YES → Go to Step 3                      │
│   └─ NO  → 传感器可能故障，降速运行          │
└────────────────────────────────────────────┘

┌─ Step 3：判断是否真实过温 ─────────────────┐
│ ☐ 环境温度<30°C?                            │
│ ☐ 近1h负载未超过50%?                       │
│ ☐ 上次成功冷却到70°C是在多久前?            │
│                                              │
│ 判定：如果环境好且负载低，但温度高 → 真实故障│
│       原因：散热器污堵 或 冷却液污染          │
└────────────────────────────────────────────┘

【处置】
  ✓ 立即停机，等待冷却至80°C以下
  ✓ 检查：散热器是否被阻挡（油漆、灰尘、垃圾）
  ✓ 检查：冷却液颜色（应为绿色，如果变成棕色→污染）
  ✓ 若5h仍无法冷却，报告技术员更换冷却液或电机


【分支 A：流量低故障】
┌─────────────────────────────────────────────┐
│ ☐ 泵电源指示灯是否点亮?                     │
│   └─ YES → 泵已上电                          │
│   └─ NO  → 检查24V电源，可能断路             │
│                                              │
│ ☐ 泵是否有运转声音(正常运转应有轻微嗡嗡声)?│
│   └─ YES → 泵转，问题在管路                  │
│   └─ NO  → 泵卡死或电源故障                  │
│                                              │
│ ☐ 泵出口压力表是否指示(应>100kPa)?         │
│   └─ YES → 流量计故障，联系技术员            │
│   └─ NO  → 泵吸不上液体                      │
│                                              │
│ 最可能原因：吸入口滤器堵塞                   │
│ 临时措施：更换滤芯（如可访问）               │
└─────────────────────────────────────────────┘
```

## 18.5 故障排除决策矩阵与根本原因分析

### 18.5.1 典型故障的根本原因分析(RCA)

#### **案例1：冷却液流量突然从100%降至20%**

**故障树构建**：

```
        流量低(Q<20%)
            │
    ┌───────┼───────┐
    ▼       ▼       ▼
  泵侧    管路    热交
  故障    堵塞    器侧
   │       │       │
   ├─①泵转速低    ├─① 进口滤器堵
   ├─②泵吸空      ├─② 管路积冰/积碳
   ├─③泵卡死      └─③ 换热器芯子堵
   └─④检查磨损
```

**诊断决策**：

```
Step1: 检查泵电源 
  → 24V电源正常? 
     YES: Go to Step2
     NO:  故障根因=电源断路
          处置=检查熔断器/接头

Step2: 检查泵转速(编码器反馈或听诊)
  → 泵是否转动?
     YES: Go to Step3  (泵吸不上液体)
     NO:  故障根因=泵轴卡死
          处置=检查液位/吸入口堵塞

Step3: 检查液位
  → 液位是否正常(应在中线以上)?
     YES: Go to Step4  (管路可能堵)
     NO:  故障根因=液体不足
          处置=补充冷却液，查漏

Step4: 检查进口压力表(在泵吸入口)
  → 压力是否<-0.5bar(负压)?
     YES: Go to Step5  (管路堵或滤器堵)
     NO:  故障根因=泵内磨损
          处置=更换泵

Step5: 检查滤芯(若可访问)
  → 滤芯是否变黑/堵塞?
     YES: 故障根因=滤芯污堵
          处置=更换滤芯
     NO:  故障根因=管路积碳/积冰
          处置=更换冷却液或清洗管路
```

**诊断证据表**：

| 症状组合 | 根本原因 | 置信度 | 快速验证方法 |
|---------|--------|------|----------|
| Q↓, P_inlet<-0.5bar, T_oil↓ | 滤芯堵塞 | 95% | 打开滤芯壳观察 |
| Q↓, P_inlet正常, T_oil↑ | 泵内磨损 | 85% | 新泵试机对比 |
| Q↓, 泵无声, I_pump=0 | 电源故障 | 99% | 测24V电压 |
| Q↓, 泵正常转, P_in<-0.5bar, 液位低 | 液体泄漏 | 90% | 检查管接头 |

#### **案例2：电机过流(I_motor > 1.5Irated)**

**根本原因树**：

```
        电机过流
         │
    ┌────┼────┐
    ▼    ▼    ▼
  机械  电气  控制
  侧    侧    侧
   │     │     │
   ├─轴系卡   ├─相序错
   ├─轴承坏   ├─缺相
   ├─螺旋桨堵 ├─母线低压
   └─对中不良 └─变频器故障

**快速诊断流程**：

```
Q1: 转速表是否显示异常?
   → 转速=0(推进轴未转):
       原因=机械卡死(螺旋桨堵或轴承坏)
       处置=停机检查螺旋桨、轴承
   
   → 转速=额定值:
       原因=电气故障或变频器故障
       处置=Go to Q2

Q2: 直流母线电压是否<700V(for 800V系统)?
   → YES: 原因=母线欠压
          处置=检查充电器、船电质量
   
   → NO: Go to Q3

Q3: 三相电流是否平衡(偏差<10%)?
   → NO: 原因=缺相或相序错
         处置=检查变频器连接、船电相序
   
   → YES: Go to Q4

Q4: 变频器LED指示是否正常(绿灯闪烁)?
   → NO: 原因=变频器内部故障
         处置=更换变频器
   
   → YES: 原因=控制逻辑故障或传感器故障
          处置=检查编码器/负载反馈
```

## 18.6 可靠性增长曲线与故障分析

### 18.6.1 MTBF 与 MTTR 计算

**定义**：

```
MTBF (Mean Time Between Failures) = 总运行时间 / 故障次数
例：一艘船年运营8000h，一年内发生2次故障
    MTBF = 8000 / 2 = 4000小时 ≈ 2个月

MTTR (Mean Time To Repair) = 总维修时间 / 故障次数  
例：两次故障分别耗时8h和12h
    MTTR = (8 + 12) / 2 = 10小时

系统可用性 = MTBF / (MTBF + MTTR) × 100%
           = 4000 / (4000 + 10) × 100%
           ≈ 99.75%
```

### 18.6.2 可靠性增长曲线(Duane Model)

**Duane公式**：

```
MTBF(累计运行时间 T) = K × T^α

其中：
  K = 初始可靠性系数
  α = 改进率参数(0.3~0.5)
  T = 累计运行时间
```

**船舶电推系统典型曲线**：

```
MTBF
(h)  │
5000 │
     │                  ╱╱╱╱
4000 │            ╱╱╱╱╱
     │        ╱╱╱╱
3000 │    ╱╱╱╱        <- α=0.4(典型改进率)
     │  ╱╱╱
2000 │╱╱
     │
1000 │  初期故障期      烧入期        稳定期
     │  (0-500h)      (500-2000h)   (>2000h)
     │
  0  └─────┬─────────┬─────────────→ 累计运行时间(h)
          500       2000          5000

解释：
• 初期故障期：新设备MTBF=1000h，故障频率高(制造缺陷)
• 烧入期：通过FAT+运维，不断发现并改进设计缺陷，MTBF逐月增长
• 稳定期：MTBF>3000h，设计和工艺基本稳定，主要是寿命末期衰减
```

**增长目标设定**：

| 项目 | Year 1 | Year 2 | Year 3 | Year 5 |
|-----|--------|--------|--------|--------|
| 目标MTBF | 2000h | 4000h | 6000h | >8000h |
| 累计故障数 | <5次 | <3次 | <2次 | <2次/年 |
| 平均故障间隔 | 1600h | 2667h | 4000h | 4000h |

### 18.7.2 故障改进案例

**案例：冷却液流量传感器频繁故障**

```
问题描述：
  • 2023年Q1：3艘船的冷却液流量传感器在海上故障
  • 故障模式：读数突然跳变或卡在某个值
  • 频率：每月1次
  • 影响：误触发流量低警报，增加船员紧张

  传感器型号：Siemens SITRANS F M MAG5100 (4-20mA输出)
  工作条件：盐水冷却液，实际流速0.5~2 m/s

  ① 电磁式传感器对海上EMI敏感
  ② 冷却液中的盐分沉积在电极→信号漂移
  ③ 4-20mA模拟信号缺乏抗干扰能力
  
  根本问题：选型不符合舰用应用环境

改进方案：
  Original:  4-20mA模拟传感器 → Siemens SITRANS F M MAG5100
  Improved:  Modbus数字传感器 → ABB COMAP C4 (RS485)
  
  优点：
  • 数字通信抗干扰能力强(≈100×)
  • 自诊断功能(可检测电极污染)
  • 可实时校准(无需停机拆卸)
  
验证效果：
  升级后3个月，流量传感器故障率从0.3/月 → 0 ✓
  MTBF从1200h → >8000h ✓
  
后续行动：
  ① 反馈至设计文档(Ch3 系统架构) → 所有新造船采用数字传感器
  ② OTA固件更新 → 流量传感器故障检测逻辑升级(自动切换至备用传感器)
  ③ 培训 → 所有运营船队进行传感器替换(18个月完成)

## 18.9 典型故障场景与应急预案

### 场景1：海上紧急停机(E-STOP失效)

**事件触发**：驾驶员按E-STOP，推进器仍在转

**应急流程**：
```
第0秒：发现E-STOP失效
  → 立即按下"机舱E-STOP"（如装备）
  → 如机舱按钮也无反应 → Go to Step1

第5秒：切换至应急停机模式(Manual E-STOP)
  Step1: 拔出主断路器(船舶24V电源)
         → 所有继电器线圈失电
         → E-STOP触点自动弹开
         → 推进器应在3秒内停止
  
  Step2: 若仍未停止，执行"柴油应急停机"
         (仅限混合动力船，需事前训练)

第30秒：系统状态评估
  ✓ 推进器已停 → 进入应急恢复流程
  ✗ 推进器未停 → 报告"系统失控"，启动船舶应急预案
```

**预防措施**：
- 驾驶员每周执行一次E-STOP功能检查
- 每月一次故障注入测试(断开E-STOP继电器，验证机舱按钮能否停机)
- FAT中明确标记E-STOP的备用停机路径

### 场景2：海上冷却系统完全失效

**事件触发**：冷却液流量突降至0，温度无法读取

**应急决策**：

```
发现时刻：T0秒

T0~T10：判定故障严重度
  ✓ 流量=0，温度传感器全灭？ → 硬件故障(继电器卡死或泵坏)
  ✓ 流量=0，但温度还在上升？ → 泵故障(电机仍有热输出)
  
  决策：无论哪种情况，停机是唯一安全选择
        (冷却系统不可恢复的情况下继续运行=火灾风险)

T10~T300：应急恢复(若配备备用冷却系统)
  ✓ 有备用独立冷却泵？
     → 自动切换(继电器打开第二泵)
     → 温度恢复正常 → 可以有限继续运行
     → 同时启动备用泵诊断(检查第一泵)
  
  ✗ 无备用泵？
     → 必须停机并靠岸
     → 等待维修或换泵

T300+：恢复与后续
  ① 修复或更换故障泵
  ② 更新冷却液(防止污染)
  ③ 执行完整功能检查(冷却、电机、控制)
  ④ 向船级社报告(可能需要增加检验项)
```

## 18.10 常见问题(FAQ)与答案

| 问题 | 答案 | 关键点 |
|-----|------|--------|
| **Q1: 为什么故障码这么多（50+个）？** | 细化故障分类有利于快速定位。每个故障码对应特定的检查清单，船员可以按图索骥。相比笼统的"系统故障"提示，具体的故障码能将排查时间从几小时降至10分钟。 | 冗余安全：宁可多报，不可漏报 |
| **Q2: 云端告警延迟会不会太高？** | 云端告警是**辅助**，不是主要决策。E-STOP等SIL2级别的安全功能完全独立，不依赖云。云的作用是记录、统计、提前告警（如温度趋势预警），给人工干预的时间。 | 架构分层：本地自主+云端支持 |
| **Q3: 故障排除手册能否减少培训时间？** | 可以。FAQ+决策树+检查单相结合，非专家船员也能在3~5分钟内完成初步诊断。但**不能替代**现场技能培训。建议模式：线上培训(看手册)+现场演练(模拟故障)。 | 知识体系化+实战训练 |
| **Q4: 如何处理"仪表板显示多个故障码"的情况？** | 按优先级处理。如"E-STOP失效(SIL2)" + "传感器故障(SIL0)"同时出现，优先处理E-STOP。决策树中会明确标出优先级，遵循规则决策。 | 单一明确的优先级标准 |

## 18.14 故障排除常见误操作与纠正

### 误操作1：无故停机

**症状**：系统报告"冷却液流量低"，但泵转、流量表显示正常

**错误做法**：停机并等待技术员，船舶延误

**正确做法**：
```
Step1: 验证流量表的可靠性
  ① 检查流量表是否有玻璃管气泡 → 有气泡需排气
  ② 比对流量表读数与控制器读数(HMI显示) → 偏差>10%表示流量表故障
  ③ 若都正常 → Go to Step2

Step2: 检查故障诊断逻辑
  ① 查询控制器日志 → "流量低"是否频繁上下跳变?
  ② 若频繁跳变 → 传感器信号不稳定，需滤波调试
  ③ 若持续稳定<70% → 真实流量低，需进一步检查

Step3: 临时措施
  ① 若流量表与控制器都显示正常 → 继续运行，监控温度
  ② 记录"虚警"事件，反馈至技术部门
  ③ 计划下次靠岸时更新流量传感器标定
```

**预防**：定期标定流量表，至少每6个月一次

### 误操作2：忽视预警信号

**症状**：温度黄灯亮起，船员继续满负荷运行

**后果**：温度持续上升 → 红灯亮(E-STOP) → 船舶失控

**正确做法**：
```
黄灯阶段(110~120°C)：
  ① 立即降速至70%功率 ← 重点
  ② 检查冷却系统工作状态
  ③ 监控温度变化（应在5分钟内开始下降）
  ④ 若温度不降 → 准备降速至50% S4模式

红灯阶段(>120°C)：
  ① 完全停机 ← 不可逆转
  ② 等待冷却至100°C以下
  ③ 检查冷却液/换热器/泵的工作状态
  ④ 维修或更换故障部件后才能重启
```

**预防培训**：船员手册中用大字体和红色标记"黄灯时降速，红灯时停机"

## 本章结语

故障诊断是新能源电推系统从**"能用"到"好用"**的关键一步。

- 设计得再完美的系统，如果船员无法在3分钟内理解故障原因，在10分钟内判断是否可以继续运行，那么运维成本会非常高昂。
